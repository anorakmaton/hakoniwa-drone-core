## ドローンシミュレータ評価項目定義（選定理由追記版）

このドキュメントでは、ドローンシミュレータの評価項目を定義します。評価項目は、シミュレーションコア性能、連携性と拡張性、開発・運用効率・利便性の3つの大分類に分けられます。各項目は、シミュレータの機能や性能を評価するための基準となります。

| 大分類 | 中分類 | 評価項目 | 選定理由 |
| :--- | :--- | :--- | :--- |
| **A. シミュレーションコア性能** | 物理モデル | 物理エンジンの種類と精度 (空気抵抗、地面効果など) | 現実世界との乖離が少ないほど、シミュレーションの信頼性が向上するため。特に空力特性の再現度は飛行安定性に直結する。 |
| A. シミュレーションコア性能 | 物理モデル | 物理エンジンの拡張性 (MuJoCo, Matlab連携など) | より高度な物理現象を扱いたい場合や、既存の研究資産を活かした連携・拡張が可能か評価するため。 |
| A. シミュレーションコア性能 | 機体モデル | 機体特性の拡張性(重量、推力、バッテリー消費など) | 多様なドローンをシミュレーション対象とするために、機体のパラメータを柔軟に設定・変更できる必要があるため。 |
| A. シミュレーションコア性能 | センサーモデル | 標準センサーの種類と精度 (カメラ, IMU, LiDAR) | 自律飛行に不可欠なセンサー群が標準で提供され、その出力が現実のセンサー特性に近いか評価するため。 |
| A. シミュレーションコア性能 | センサーモデル | カスタムセンサーの拡張性 | 研究開発段階の特殊なセンサーなど、標準以外のセンサーを独自に実装・テストできるか評価するため。 |
| A. シミュレーションコア性能 | 環境モデル | 3D環境の再現度と拡張性(自作・編集の自由度) | 現実の飛行環境を模したリアルな環境（デジタルツイン）でテストを行うため。また、独自のテストコースを作成できる自由度も重要。 |
| A. シミュレーションコア性能 | 環境モデル | 外部環境の拡張性(風、気圧、障害物など) | 悪天候や非定常な状況に対するドローンの堅牢性をテストするため。現実世界の不確実性をシミュレーションに反映できるかが重要。 |
| **B. 連携性と拡張性** | FC連携 | フライトコントローラの対応範囲 (Ardupilot/PX4/自作) | 実機で利用するFCと連携(SIL/HIL)できなければ、制御アルゴリズムの有効な検証ができないため。 |
| B. 連携性と拡張性 | FC連携 | 通信プロトコル対応 (Mavlinkなど) | FCやGCSとの通信には標準プロトコルが必須であり、その対応状況はシステムの連携性を左右するため。 |
| B. 連携性と拡張性 | 外部API連携 | プログラミング言語対応 (Python, C++等)とAPIの質 | 自動テストや強化学習など、シミュレーションをプログラムから制御するために、使いやすく高機能なAPIが提供されているかが重要。 |
| B. 連携性と拡張性 | 外部API連携 | ROS/ROS2への対応レベル | ロボット開発における標準的なフレームワークであるROS/ROS2との親和性は、開発効率に大きく影響するため。 |
| B. 連携性と拡張性 | 外部システム連携 | 他システムとの連携実績 (デジタルツイン/MCPなど) | 大規模なシステム（群制御プラットフォーム等）の一部として利用できるか、その実績を評価するため。 |
| **C. 開発・運用効率・利便性** | スケーラビリティ | マルチビークル対応 (同時実行数) | ドローンの群制御アルゴリズムを開発・検証するために、多数の機体を同時にシミュレーションできる性能が必要なため。 |
| C. 開発・運用効率・利便性 | スケーラビリティ | 高速化対応(ヘッドレスモードの実行速度) | 機械学習など大量の試行回数が必要な場合に、描画を省略した高速なシミュレーションが可能か評価するため。 |
| C. 開発・運用効率・利便性 | 開発支援 | デバッグ機能(ログ記録/リプレイ) | 問題発生時に原因を特定しやすくするため。ログの記録や特定の状態を再現する機能はデバッグ効率を大幅に改善する。 |
| C. 開発・運用効率・利便性 | 開発支援 | ビジュアライズ機能の選択・分離の可否 | 用途に応じて、デモ用の高品質な描画と開発用の軽量な描画を切り替えられる柔軟性があるか評価するため。 |
| C. 開発・運用効率・利便性 | サポート体制 | 公式ドキュメントの質と量 | 開発者が使い方を学習し、問題解決を行う上で、充実した公式ドキュメントは不可欠なため。 |
| C. 開発・運用効率・利便性 | サポート体制 | 開発者コミュニティの活発さ | 公式ドキュメント以外の情報収集や、開発者間の情報交換が活発であるほど、エコシステムが健全に発展している証拠となるため。 |
| C. 開発・運用効率・利便性 | 実行環境 | 対応OS (Windows/Linux/Mac) | 開発者が利用する多様なOSに対応していることで、導入の障壁が下がり、より多くのユーザーが利用できるため。 |
| C. 開発・運用効率・利便性 | 実行環境 | 要求ハードウェアスペック | 利用にあたり高価な機材が不要であれば、個人や教育機関でも導入しやすくなるため。 |
| C. 開発・運用効率・利便性 | 実行環境 | 特殊環境対応(モバイル, VR/ARデバイス等) | VRでの没入型操縦訓練や、ARでの現実空間との重ね合わせなど、新たなユースケースへの対応力を評価するため。 |
| C. 開発・運用効率・利便性 | アーキテクチャの柔軟性 | シミュレーションアーキテクチャの分離性とモジュール性 | 物理エンジンやレンダリングエンジンなど、必要な機能だけを分離して利用できると、他のシステムへの組み込みが容易になるため。 |
| C. 開発・運用効率・利便性 | 地域コミュニティのアクセシビリティ | 地域言語サポートとローカルコミュニティの充実度 | 母国語のサポートや地域のコミュニティが存在することで、導入や活用のハードルが下がり、普及が促進されるため。 |
| C. 開発・運用効率・利便性 | 教育用途への最適化 | 教育用途での利用しやすさ | 初学者がドローンやプログラミングを学ぶ教材として、導入の容易さや分かりやすいサンプルが提供されているか評価するため。 |
| C. 開発・運用効率・利便性 | デモ用途のプレゼンテーション性 | デモ用途での視覚的訴求力とインタラクティブ性 | 研究成果や製品をアピールする際に、見栄えの良いグラフィックスや直感的な操作性は、理解を助け、強い印象を与えるため。 |
| C. 開発・運用効率・利便性 | 連携と利用の柔軟性 | 独立したライブラリとしての利用可否と実行環境の多様性 | 既存のシミュレータの枠を超えて、独自のアプリケーションに物理演算やセンサーモデルを組み込めるか、といった高度な利用を想定するため。 |
